# Forge AI

Forge AI is the AI-DOS Development and Autonomy Enablement Target Project. It develops, validates, self-applies, and hardens AI-DOS through bounded, Human-Governed work with explicit evidence.

This README is a repository overview and navigation aid only. It is **not** the repository contract, does not replace `AGENTS.md`, and does not redefine any governed authority.

## Primary Entry Points

Use the following entry points for Forge AI Target Project work in this repository:

```text
AGENTS.md
    ↓
docs/Projects/ForgeAI/Mission/AGENTS.md
    ↓
docs/Projects/ForgeAI/Planning/ProjectStatus.md
    ↓
docs/Projects/ForgeAI/Planning/DevelopmentPhases.md
    ↓
docs/Projects/ForgeAI/Planning/Roadmap.md
```

Additional task-specific routing:

- Use `docs/Projects/ForgeAI/Mission/ForgeAI-Mission-and-Autonomy-Model.md` when mission or autonomy alignment is required.
- Use `docs/AI/` only for AI-DOS product truth and reusable capability artifacts.
- Use `docs/Projects/ForgeAI/Reports/` when reviewing Forge AI Target Project evidence records.

## Authority Boundaries

- `AGENTS.md` is the repository entry point for AI assistants.
- `docs/Projects/ForgeAI/Mission/AGENTS.md` is the canonical Forge AI Target Project contract.
- `docs/Projects/ForgeAI/Planning/ProjectStatus.md` is the Target Operational Entry and live operational-state authority.
- `docs/Projects/ForgeAI/Planning/DevelopmentPhases.md` defines Forge AI capability development phases.
- `docs/Projects/ForgeAI/Planning/Roadmap.md` defines Forge AI capability roadmap direction.
- `docs/AI/` contains AI-DOS product truth and must stay separate from Forge AI Target Project truth.

## Target-Project Portability

For Forge AI self-application tasks, use the Forge AI Target Project paths listed above in this repository.

For another Target Project that adopts AI-DOS, resolve that project's operational state, roadmap, resources, constraints, and protected areas through that Target Project's own authority chain. Do not assume Forge AI repository paths are universal for every Target Project.

## Working Rules for AI Agents

1. Start with `AGENTS.md`.
2. Read the canonical Forge AI Target Project contract before classifying, planning, editing, validating, reviewing, or reporting work.
3. Resolve continuation work through `docs/Projects/ForgeAI/Planning/ProjectStatus.md`.
4. Read only task-specific authorities after classification.
5. Preserve protected areas, roadmap, DevelopmentPhases, ProjectStatus, and AI-DOS product-purity boundaries.
6. Validate changes and report completion evidence.
