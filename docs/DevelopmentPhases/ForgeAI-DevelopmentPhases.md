## 1. Purpose

This document defines the active phase and stage sequence for development of the Forge AI repository as a self-hosted Target Repository.

It does not describe AI-DOS internal structure and does not depend on AI-DOS internal document paths.

The Target Repository root `AGENTS.md` declares:

- the AI-DOS Framework Provider entry point;
- the active ProjectStatus resource;
- the active DevelopmentPhases resource;
- the active roadmap resource;
- architecture resources;
- source roots;
- validation commands; and
- protected areas.

This document consumes those declarations after they are resolved. It does not own or duplicate them.

## 2. Target Project Boundary

This document owns only the Forge AI Target Project phase sequence.

It does not own:

- AI-DOS Framework authority;
- AI-DOS boot or internal routing;
- project-resource path declarations;
- Target Repository resolution;
- live operational state;
- roadmap direction;
- architecture truth;
- implementation authorization; or
- certification.

External Target Repositories define their own phase resources through their own root `AGENTS.md` and do not inherit this phase sequence.

## 4. Current Phase

The current phase is **Phase 9 — AI-DOS Repository Architecture & Disposition Audit v2**.

The objective is to classify the current repository after the AI-DOS / Target Repository separation, target-resolution work, path normalization, physical roadmap relocation, and self-hosting governance realignment.

This phase does not execute cleanup, move files, declare Repository Freeze, or activate Axis Suite work.

## 5. Current Stage

**Current stage:** complete the definitive repository architecture and disposition audit.

**Required result:** one current classification and disposition baseline that becomes the sole cleanup execution reference while preserving earlier audits as historical evidence.

**Next stage:** revised cleanup execution, only after audit acceptance.

## 6. Phase 9 Completion Criteria

Phase 9 is complete only when:

1. The current physical repository tree has been inspected.
2. Major families are classified by owner, role, lifecycle, and location.
3. Every family has one primary disposition.
4. AI-DOS Framework truth and Forge AI Target Project truth remain separate.
5. Self-hosting evidence is not generalized as an external-project requirement.
6. Cleanup batches, dependencies, risks, and blockers are explicit.
7. Repository Freeze readiness and Axis Suite readiness are stated without activation.
8. Human Governance accepts the audit as the cleanup baseline.

## 7. Future Milestones
