## 2. Entry-Point and Resource Boundary

The Target Repository root `AGENTS.md` owns:

- the Framework Provider declaration;
- the AI-DOS Framework entry point;
- the active ProjectStatus resource;
- the active DevelopmentPhases resource;
- the active roadmap resource;
- project architecture resources;

- validation commands;
- protected areas; and

AI-DOS owns its own internal boot and authority routing behind its single entry point.

ProjectStatus consumes the resolved project context and AI-DOS operating rules. It does not:

- declare project-resource paths;
- resolve project resources;
- directly reference AI-DOS internal documents;
- provide fallback paths;
- override root `AGENTS.md`; or
- act as a project-resource registry.

## 10. Blocker Policy

Stop and report a blocker when:

- the Target Repository cannot be resolved;
- the Framework Provider entry point is unavailable;
- root `AGENTS.md` declarations are missing or ambiguous;
- required project resources cannot be resolved;
- ownership between AI-DOS Framework truth and Target Project truth is unclear;
- a disposition would modify a frozen area without authorization;
- evidence is insufficient for a safe disposition; or
- an external Target Repository would inherit Forge AI self-hosting assumptions.

ProjectStatus records blockers but does not resolve authority conflicts.

## 11. Status Update Policy

ProjectStatus updates are allowed only when:

1. Human Governance explicitly requests an update.
2. The active task is a ProjectStatus or ProjectStateUpdater task.
