## 1. Executive Summary

This roadmap governs development of the Forge AI repository as the first self-hosted Target Repository of Forge AI / AI-DOS.

The Target Repository depends only on:

- its own root `AGENTS.md`;
- the AI-DOS Framework entry point declared there; and
- its own resolved project resources.

It does not depend on AI-DOS internal file paths.

The next objective is to complete one definitive repository audit, execute its cleanup dispositions, freeze the normalized baseline, and prove external operation through Axis Suite.

## 5. Active Program — Repository Architecture & Disposition Audit v2

The active program shall:

- inspect the current repository;
- distinguish reusable AI-DOS Framework truth from Forge AI Target Project truth;
- classify active, frozen, historical, legacy, archive, and removal surfaces;
- provide one primary disposition per family;
- identify the final physical layout;
- define executable cleanup batches;
- assess Repository Freeze readiness;
- assess Axis Suite readiness; and
- preserve prior audits as historical evidence.

It shall not execute cleanup or modify audited source documents.

## 6. Cleanup Program

Cleanup begins only after audit acceptance.

Rules:

1. Use the final audit as the sole execution baseline.
2. Do not re-audit during execution batches.
3. Do not redesign architecture.
4. Preserve current owners and frozen areas.
5. Merge unique content before removing superseded sources.
6. Separate historical evidence from active authority.
7. Update inbound references in the same batch as moves.
8. Preserve the single AI-DOS Framework entry-point boundary.
9. Record blockers instead of inferring dispositions.

## 7. Repository Freeze Program

Repository Freeze requires:

- completed cleanup batches;
- resolved active references;
- unambiguous Framework and Target Project ownership;
- no duplicate active authority;
- internally consistent frozen families;
- deterministic target resolution and boot;
- planning documents matching repository state; and
- explicit Human Governance approval.

Freeze does not certify Runtime implementation, Engine implementation, or commercial readiness.

## 8. Axis Suite Preparation and Pilot

Axis Suite must declare through its own root `AGENTS.md`:

- AI-DOS Framework Provider entry point;
- Axis Suite ProjectStatus resource;
- Axis Suite DevelopmentPhases resource;
- Axis Suite roadmap;
- architecture resources;
- source roots;
- validation commands;
- protected areas; and
- project authority order.

Axis Suite must not reference AI-DOS internal document paths.

The first pilot must exercise the full operating flow and complete a bounded real task safely and repeatably.

## 9. Evidence-Driven Improvement Rule

After the pilot, changes must be justified by observed evidence such as:

- missing declarations;
- ambiguous ownership;
- unsafe fallbacks;
- unnecessary context;
- failed validation routing;
- repeated manual intervention; or
- measurable onboarding friction.

Pilot evidence may justify focused changes. It does not justify reopening the entire architecture.

## 10. Productization Gate

Branding, installation, distribution, onboarding automation, commercial packaging, and pricing remain deferred until AI-DOS demonstrates:

1. successful self-hosting;
2. successful Axis Suite operation;
3. repeated external-project workflows;
4. deterministic onboarding;
5. stable validation and blocker behavior; and
6. a user-facing path that depends only on entry points, not internal documentation knowledge.

## 11. Safety Rules

- Target Project documents must not directly reference AI-DOS internal document paths.
- AI-DOS must expose one Framework Provider entry point.
- Root `AGENTS.md` must own provider and project-resource declarations.
- Do not create foundational architecture without a proven unresolved problem.
- Do not execute cleanup before audit acceptance.
- Do not declare Repository Freeze from roadmap language alone.
- Do not begin Axis Suite work before repository readiness.
- Do not call AI-DOS operationally proven before an external pilot succeeds.
- Prefer execution and evidence over speculative documentation.
