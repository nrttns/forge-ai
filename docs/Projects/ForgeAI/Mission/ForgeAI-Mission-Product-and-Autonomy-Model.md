# Forge AI Mission, Product, and Autonomy Model

Version: 2.0.0-draft

## Purpose
Forge AI is the Product Development Target Project for AI-DOS.

Forge AI develops, validates, distributes, operates and evolves AI-DOS as a reusable AI Operating System.

## Identity
- Forge AI: product-development Target Project.
- AI-DOS: reusable AI Operating System.
- Target Projects: independent consumers.

## Product Lifecycle
Private Development Repository
→ Build
→ Versioned Distribution
→ Local CLI
→ Local MCP
→ Hosted Provider
→ Independent Target
→ Execution Evidence
→ Capability Evolution
→ Next Release

## Mission
1. Develop AI-DOS.
2. Preserve product purity.
3. Produce governed distributions.
4. Validate reusable capabilities.
5. Learn from Target evidence.
6. Govern capability evolution.
7. Prove external Target readiness.
8. Sustain AI Highway operations.

## Product Boundaries
Forge AI owns planning, evolution and releases.
AI-DOS owns runtime behavior and public contracts.
Target Projects own mission, planning, code and validation.

## Self-Application
Forge AI is the first dogfooding Target.
Independent Target evidence is equally valid.

## Capability Evolution
Target Execution
→ Evidence
→ Opportunity
→ Human Governance
→ Bounded Product Change
→ Validation
→ Release

## Distribution
v1 Local CLI
v2 Local MCP
v3 Managed Provider

## Safety
Human Governance is final.
No Target contamination.
No automatic release.
No automatic capability promotion.
Product truth remains private.

## Success
AI-DOS is distributable, reusable, Target-independent and evolves through governed evidence.
