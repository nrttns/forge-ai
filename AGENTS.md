## 3. Repository Interpretation Boundary

The presence of `package.json`, npm scripts, TypeScript, JavaScript, `src/`, `dist/`, tests, or CI workflows shall not be interpreted as evidence that:

- Forge AI or AI-DOS is fundamentally a JavaScript, TypeScript, Node.js, or npm product;
- `src/main.ts` is the AI-DOS engine, kernel, architecture, or normative system definition;
- implementation technology owns product or Target truth;
- an existing implementation language is preferred for future work;
- passing implementation-specific checks validates the whole AI-DOS system.

Current JavaScript and TypeScript artifacts are bounded implementation surfaces, including the accepted D2 Local CLI distribution and bootstrap surface. They are replaceable and subordinate to the governing Markdown contracts.

Agents must derive architecture, authority, scope, implementation direction, and validation claims from the mandatory governance chain and the active bounded task, never from file extensions, package metadata, build tooling, repository-language statistics, or proximity to existing code.
